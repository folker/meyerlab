MG-RAST v5

Implement a comprehensive next generation microbiome pipeline that includes clustering, assembly and MAG generation. This is a collaborative project with the MG-RAST team at the University of Chicago and Argonne National Laboratory in Chicago, Illinois. MG-RAST is a hosted microbiome science platform that uses an efficient pipeline executed distributed resources (clouds, grids and clusters) to process microbiome data of various types. Data is then integrated into a data science platform for web based visualization, comparison and analysis. Awe

Awe is a light weight workflow manager for scientific computing at scale that executes CWL workflows.

It is a cloud native workflow platform designed from the ground up to be fast, scalable and fault tolerant. Awe is RESTful. The API is accessible from desktops, HPC systems, exotic hardware, the cloud and your smartphone

visit homepage

Shock

Shock is a platform to support scientific computation, storage, and distribution.

Designed from the ground up to be fast, scalable, fault tolerant, and implemented as a thin layer atop other storage paradigms. Shock is RESTful and allows storage and querying of complex metadata. Shock is a data management system. The long term goals of Shock include the ability to annotate, anonymize, convert, filter, perform quality control, and statically subsample at line speed bioinformatics sequence data. Extensible plug-in architecture is in development.

A Shock server supports remote data storage and local caching and is currently used in production with the following remote stores:

  • Shock (Type: Shock)
  • S3 (Type: S3)
  • IBM Tivoli TSM (Type: TSM)

    • Shock provides protoypical support for:

  • Amazon Glacier
  • Google Cloud Storage
  • Microsoft Azure Data Storage

  • DAOS
  • Globus
  • IRoDS

    Mikrobiomdaten stellen durch Größe und Komplexität signifikante Herausforderungen bei der Analyse dar. Im Rahmen des Fachprojekts führen wir die bioinformatische Analuse von realen Mikrobiom-Datensätzen durch. Ziel ist es Researchobjects zu erzeugen, welche die einzelnen Analyseschritte und Ergebisse widerspiegeln. Dazu nutzen wir CWL und CWLProv. Wir werden auf existierenden Rechnern (Clustern, Grids und Clouds) aus den Sequenzdaten Informationen für die Interpretation der Daten erstellen und Visualisierungen erstellen.

    Die Aufgabe der Studendierenden ist, auf Basis von aktueller Literatur, eine allgemein verwendbare, effiziente Implementierung einer Pipeline zur Genomanalyse in CWL zu erstellen.
